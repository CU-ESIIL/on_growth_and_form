# Environmental Data Science Stack

This page summarizes a broader conceptual framework for environmental data science as a layered scientific intelligence system. It belongs in `background_context/` because it is strategic framing, not sponsor-controlled guidance.

## Why it matters here

This framework is relevant to proposal positioning because it helps explain how wildfire modeling, data assimilation, AI agents, and decision support can fit into one coherent research architecture.

## Stack summary

1. Planetary data substrate
2. Data harmonization and semantic layer
3. Continuous modeling layer
4. AI scientific agents
5. Scientific orchestration layer
6. Interactive scientific interfaces
7. Decision and governance layer

## Working implications for this repo

- `simulations/` can be framed as the continuous modeling layer
- `citations/` and background notes can support literature and theory agents
- proposal narrative can position the work as part of a continuously operating scientific intelligence system
- figures can help visualize the stack from data to decision support
