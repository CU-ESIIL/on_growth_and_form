# FIRED and CubeDynamics

This page summarizes a systems view of event-based Earth system analysis built around wildfire event delineation and spatiotemporal data cubes.

## Why it matters here

If the proposal aims to study wildfire growth, perimeter geometry, or scaling across many events, then an event-based analytical infrastructure is essential. `FIRED` and `CubeDynamics` provide one way to move from raw burned pixels to structured wildfire events embedded in a broader Earth system context.

## Core ideas

- `FIRED` turns burned-area pixels into coherent wildfire events
- `CubeDynamics` organizes environmental data as spatiotemporal cubes
- together they connect event reconstruction with scalable analysis
- this combination supports geometry, scaling, and environmental-driver studies across many fires

## Relevance to proposal framing

- supports arguments for event-based wildfire analysis rather than isolated pixel processing
- links wildfire geometry research to scalable Earth system data infrastructure
- provides a framework for integrating fire growth, vegetation, climate, and fuel context in one analytical system
