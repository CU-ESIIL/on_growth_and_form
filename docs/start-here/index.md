## What this section is for

This is the orientation layer for the website. It is where first-time visitors can decide whether they need the proposal case, the scientific idea, the execution plan, or the repository workflow.

## Best first stops

- **Need the high-level proposal argument?** Start with the [Project Overview](../project-overview/index.md).
- **Need the scientific claim?** Go to the [Theory Overview](../theory/index.md).
- **Need the executable plan?** Read the [Research Program Overview](../research-program/index.md).
- **Need the workspace logic?** Open the [Proposal Workflow](../workflow.md) and [Repository Map](../repository-map.md).

## Reader pathways

### Reviewers

Start with the [Project Overview](../project-overview/index.md), then read [Program Context](../project-overview/program-context.md) and the [Research Program](../research-program/index.md).

### Collaborators

Start with the [Resubmission Strategy Document](../project-overview/review-resubmission-strategy.md), then use the [Proposal Workflow](../workflow.md) and [Repository Map](../repository-map.md) to understand how work is organized.

### Theory readers

Start with the [Theory Overview](../theory/index.md), then read [On Growth and Form](../theory/on-growth-and-form.md) and the [Wildfire Scaling Hypothesis](../theory/wildfire-scaling-hypothesis.md).

### Technical readers

Start with [Data and Infrastructure](../data-infrastructure/index.md), then move to [Methods](../methods/index.md) and [Models](../models/index.md).
