# Methods

This section turns the project's ideas into measurable procedures.

## Methods system summary

This methods system is organized as a discovery pipeline in which each stage takes a concrete action, produces measurable outcomes, and uses those outcomes to eliminate incorrect explanations until only mechanisms consistent with all independent constraints remain.

The canonical sequence in this repository is:

- `Method 1`: Measurement, Validation, and Fire-Environment Coupling
- `Method 1B`: Velocity Evaluation
- `Method 1C`: Model-Data Comparison and Scoring Framework
- `Method 2`: Generative Modeling and Experimental Design
- `Method 3`: Synthetic Validation, Uncertainty, and Identifiability

Taken together, this forms a closed loop: define what is real, remove misleading quantities, define how models are judged, generate candidate mechanisms, and verify that the judging system is reliable. In that sense, the methods are not a loose collection of analyses, but a linked sequence of discovery steps in which each later stage depends on constraints established by the earlier ones.

## Why this section matters

The proposal is not only about a theory of wildfire geometry. It is also about how that theory can be tested. The methods pages describe the analytical steps that connect event reconstruction to scaling claims.
