# Funder Materials

This section is the working home for materials that keep the proposal pointed at the actual FIRE-MODEL call. It includes the briefing you assembled about the funder, the current checklist of required components, and the paths where official solicitation files and templates should live.

## What belongs here

`funder/` is for the things you use when asking questions like:

- What is this call trying to fund?
- What documents are required?
- What should we verify against the solicitation before writing?
- Which pages or links do we need in view while drafting?

In this repo, that means three kinds of content:

- funder-oriented guides such as the FIRE-MODEL briefing
- requirement and compliance summaries
- official source documents and templates when they are available

## Current funder-facing files

- `funder/solicitation/fire_model_2026_briefing.md`
- `funder/review_criteria/fire_model_requirements_checklist.md`
- `funder/templates/`

## How this differs from background context

The `background_context/` section is for scientific framing and synthesis. It helps answer questions like:

- What theories help explain wildfire form and growth?
- How should we talk about scaling, geometry, and interface dynamics?
- What data-system architecture supports the analysis?

That material is important, but it should not be confused with what the funder is explicitly asking for.

## Working rule

When there is tension between a compelling scientific framing and the call itself, the call wins. Use this section to keep that priority visible.
