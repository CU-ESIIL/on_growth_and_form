# Program Context

This page is the bridge between the funder-facing FIRE-MODEL briefing and the broader scientific background notes. It explains how the program context shapes the way this repository frames the proposal.

## What this section is for

The `funder/` pages tell us what the FIRE-MODEL call appears to prioritize. This page translates that into a more narrative question:

What kind of scientific story is this proposal trying to tell inside the structure of the call?

## Current working interpretation of FIRE-MODEL

The program context notes point toward a proposal that is:

- explicitly cross-scale, from ignition and spread to larger landscape dynamics
- interdisciplinary by design rather than by decoration
- computational in implementation, but not purely computational in purpose
- expected to connect modeling with observational or empirical validation
- framed for practical usefulness in prediction, management, or decision support

## Why this belongs in background context

This is not a compliance page and not a source-document page. It is a positioning page. Its job is to help the proposal team translate the call into a coherent scientific argument and avoid drifting into either generic wildfire language or purely technical modeling language.
